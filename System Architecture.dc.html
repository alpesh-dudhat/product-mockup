PromoWear Mockup Builder — System Architecture
Stack Data Models IndexedDB Modules State Flow PDF Engine
Browser-Only · No Backend Required

System Architecture

Full client-side application. All data stored in IndexedDB. No server, no login, no upload limits.

Technology Stack

🎨
Fabric.js 6
Canvas
Interactive canvas engine. Handles drag, resize, rotate, crop, perspective transform on logos. Serialises full canvas state to JSON for IndexedDB storage.
cdn.jsdelivr.net/npm/fabric
✨
@imgly/background-removal
WASM
Runs entirely in-browser via WebAssembly. No API key, no cost. Removes logo/image backgrounds with AI quality. ~2–5s processing time.
cdn.jsdelivr.net/npm/@imgly/background-removal
📄
jsPDF + html2canvas
PDF
html2canvas renders each Fabric.js canvas to a high-res PNG (2×/3× scale). jsPDF composes the final PDF with chosen layout (1/2/3/4-up, horizontal/vertical).
cdnjs · unpkg
🗄
IndexedDB (idb)
Storage
Native browser database. Stores images as raw Blobs (no base64 bloat), canvas JSON, form data and session metadata. Effectively unlimited storage.
npm/idb — thin promise wrapper
📦
JSZip
Export
Bundles draft data + images into a downloadable .zip file for the "Download backup" feature. User re-imports the zip to restore a session on any device.
stuk.github.io/jszip
⚛
Vanilla JS + DC Runtime
Core
Built as a Design Component (.dc.html). No build step, no bundler. External libraries loaded from CDN via <script> in <helmet>. State managed in the DCLogic class.
Mockup Builder.dc.html

IndexedDB Schema

DB name: promowear_mockup  ·  version: 1
store: sessions
keyPath: "id"
index: updatedAt
Stores Session objects. Sorted by updatedAt for recent-sessions list.
store: items
keyPath: "id"
index: sessionId
One record per product item. canvasJSON can be large — stored as string, not object.
store: assets
keyPath: "id"
index: mimeType
Blob stored natively — no base64. BG-removed version replaces original blob in-place.
Auto-save strategy
1.Form fields → debounced 800ms after last keystroke → write Item to IDB
2.Canvas moves/resizes → debounced 1500ms → serialize canvas.toJSON() → write Item to IDB
3.Image/logo upload → immediate → write Asset blob to IDB → update Item.logoIds
4.Tab switch → flush any pending debounced writes immediately

Module Breakdown

AppShell
Layout
Top bar (logo, save, export). Item tab bar. Renders active item view. Manages session state and tab switching.
state: { session, activeItemId }
CanvasEditor
Core
Wraps Fabric.js canvas. Handles garment image loading, logo placement objects, transform tools (rotate, crop, perspective), BG colour/image, resize-without-quality-loss.
fabricCanvas, selectedObject
CanvasToolbar
UI
Toolbar above canvas. Shirt tools (resize, BG). Logo tools (rotate, scale, crop, perspective, remove BG, delete). Enabled/disabled based on selection state.
props: { selected, onAction }
DetailsPanel
UI
Right-side scrollable panel. Three sections: Logos (upload + chip list), Product Details (all fields), Artwork & Decoration. Auto-saves on change.
props: { item, onChange }
CopyFromBar
Feature
Amber banner shown on items 2+. Checkboxes: same logos, same position, same size, same details, same decoration. On tick: copies relevant data from item[0] (or last item).
props: { sourceItem, onCopy }
BgRemovalService
WASM
Lazy-loads @imgly/background-removal on first use. Accepts asset Blob, returns new Blob with transparent background. Updates Asset in IDB, reloads on canvas.
removeBackground(blob) → Blob
StorageService
IDB
All IndexedDB reads/writes via idb library. Methods: saveSession, saveItem, saveAsset, loadSession, loadItems, loadAsset, exportZip, importZip.
singleton, promise-based
PDFExportModal
Export
Modal UI: client info, layout picker (H/V + 1-4 per page), live mini-preview, extra notes. On confirm: triggers PDFEngine with settings.
state: { settings, isOpen }
PDFEngine
Export
Renders each item canvas at 3× via html2canvas → PNG. Composes A4 PDF via jsPDF using chosen layout. Adds header (PromoWear + client logos), footer, notes, page numbers.
generatePDF(session, settings)

Draft Save & Restore

Auto-save (always on)
Form edits → debounce 800ms → IDB write
Canvas changes → debounce 1500ms → IDB
Status badge: "Saved" / "Saving…"
Download backup (manual)
JSZip: session.json + items.json + /assets/ blobs → promowear-draft.zip
Re-import: drag zip onto app → restore all

PDF Engine — Layout Matrix

Layout Card orientation Grid Image area Best for
1-up · H Horizontal 1 × 1 ~45% page width Premium single-product presentation
1-up · V Vertical 1 × 1 ~50% page height Wide garments, full-width hero
2-up · H Horizontal 1 × 2 (stacked) ~38% card width Two products, still readable
2-up · V Vertical 2 × 1 (side by side) full card width Colour variant comparison
3-up · H/V Vertical 3 × 1 full card width Medium orders, compact overview
4-up · H/V Vertical 2 × 2 full card width Large orders, sign-off sheet
Render pipeline
1. Mount hidden off-screen Fabric canvas at 3× resolution
2. Load garment image + logo positions from Item.canvasJSON
3. html2canvas({ scale: 3 }) → PNG Blob
4. jsPDF.addImage(png, 'PNG', x, y, w, h) per product
5. jsPDF.addPage() when page is full
6. Draw header/footer text + logo images on each page
7. doc.save('PromoWear-Mockup-QT-xxxx.pdf')
